**1. Update config.ts:**

Replace the `gemini` and `fda` endpoint URLs with proxy paths:
```typescript
apiEndpoints: {
    geminiProxy: '/api/proxy/analyze',
    fdaProxy: '/api/proxy/fda/drug',
    rxNorm: 'https://rxnav.nlm.nih.gov/REST',
    clinicalTrials: 'https://clinicaltrials.gov/api/v2/studies',
    europePmc: 'https://www.ebi.ac.uk/europepmc/webservices/rest/search',
}
```
Remove the old `gemini` and `fda` keys. Keep the public API URLs (rxNorm, clinicalTrials, europePmc) since those don't use API keys.

**2. Simplify services/secretManager.ts:**

Remove all secret-fetching code:
- Delete `secretsCache`, `BACKEND_URL`, `fetchSecret()`, `getGeminiApiKey()`, `getFdaApiKey()`
- Keep only `checkBackendHealth()` and `checkBackendConfig()` (these are still useful)
- Remove the export of `getGeminiApiKey` and `getFdaApiKey`

The file should only contain the two health/config check functions.

**3. Refactor services/geminiService.ts — FDA data fetching:**

In `fetchFdaData()`:
- Remove the `getFdaApiKey()` call and `cachedFdaApiKey` variable
- Remove the `addApiKey()` helper function
- Change all FDA API URLs from `https://api.fda.gov/drug/...` to `/api/proxy/fda/drug/...`
- The proxy route handles key injection server-side, so URLs are simpler:
  ```typescript
  const fdaProxyBase = '/api/proxy/fda/drug';
  const labelUrl = `${fdaProxyBase}/label.json?search=${labelSearchQuery}&limit=1`;
  const totalEventsUrl = `${fdaProxyBase}/event.json?search=${eventSearchQuery}&count=safetyreportid.exact&limit=0`;
  const topReactionsUrl = `${fdaProxyBase}/event.json?search=${eventSearchQuery}&count=patient.reaction.reactionmeddrapt.exact&limit=50`;
  ```
- No `api_key` parameter needed — the proxy adds it server-side

**4. Refactor services/geminiService.ts — Gemini AI call:**

Replace the direct `@google/genai` SDK call with a fetch to the proxy:
- Remove `import { GoogleGenAI, Type } from "@google/genai";`
- Remove `let ai: GoogleGenAI | null = null;`
- Remove the `getGeminiApiKey()` call
- Replace the `ai.models.generateContent()` block (around lines 482-495) with:
  ```typescript
  const proxyResponse = await fetch('/api/proxy/analyze', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
          prompt: prompt,
          response_schema: analysisResultSchema,
      }),
  });

  if (!proxyResponse.ok) {
      const errorData = await proxyResponse.json().catch(() => ({}));
      throw new Error(errorData.error || 'Analysis temporarily unavailable');
  }

  const geminiResponse = await proxyResponse.json();
  ```
- Extract the text from the Gemini REST API response format. The response structure from the REST API is: `{ "candidates": [{ "content": { "parts": [{ "text": "..." }] } }] }`. So extract:
  ```typescript
  const jsonText = (geminiResponse.candidates?.[0]?.content?.parts?.[0]?.text || '').trim();
  ```
- The rest of the function (JSON parsing, score calculation) stays the same

**5. Remove the `Type` import and replace schema references:**

Since we're no longer using `@google/genai`, the `Type.OBJECT`, `Type.STRING`, etc. enums are unavailable. Replace them with raw string values that the Gemini REST API accepts:
- `Type.OBJECT` -> `"OBJECT"`
- `Type.STRING` -> `"STRING"`
- `Type.NUMBER` -> `"NUMBER"`
- `Type.ARRAY` -> `"ARRAY"`

This applies to the `analysisResultSchema` object definition (around lines 391-479).

**6. Handle proxy errors gracefully:**

Wrap the proxy fetch in try/catch. On error, show user-friendly message per CONTEXT.md:
- "Analysis temporarily unavailable" for Gemini failures
- For FDA failures, the existing `fetchApi` error handling works — just make sure errors from the proxy are surfaced the same way

**IMPORTANT:** Do NOT remove the `@google/genai` package from package.json in this plan. That can be cleaned up later. Just remove the import from geminiService.ts.
