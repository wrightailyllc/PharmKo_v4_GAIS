**1. Configure environment-specific CORS (SEC-04):**

Replace `CORS(app, supports_credentials=True)` with:
```python
ALLOWED_ORIGINS = os.environ.get("ALLOWED_ORIGINS", "http://localhost:5000,http://localhost:5173").split(",")
```

This allows loose origins in dev (default) and strict in production (set via `--set-env-vars` in cloudbuild.yaml). The Cloud Run URL (`https://pharmko-app-460663669978.us-central1.run.app`) will be configured as ALLOWED_ORIGINS in the deploy step.

**2. Remove /api/secrets/* endpoints (SEC-01):**

Delete the `get_gemini_key()` and `get_fda_key()` route handlers entirely (lines ~115-134). These expose raw API keys to the browser.

**3. Create Gemini proxy route (SEC-01):**

Add a new route `POST /api/proxy/analyze` that:
- Accepts JSON body with `prompt` (string) and `response_schema` (object) fields
- Reads `GEMINI_API_KEY` from environment via `get_secret()`
- Calls the Gemini REST API: `POST https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key={api_key}`
- Request body: `{"contents": [{"parts": [{"text": prompt}]}], "generationConfig": {"responseMimeType": "application/json", "responseSchema": response_schema, "temperature": 0.1}}`
- On success: returns the Gemini response JSON to the frontend
- On error: returns `{"error": "Analysis temporarily unavailable"}` with status 503 — no raw error details or stack traces per CONTEXT.md decision
- Uses `requests.post()` (already in requirements.txt) — no new dependencies needed
- Add a 120-second timeout on the requests.post() call since Gemini can be slow

**4. Create FDA proxy routes (SEC-01):**

Add a new route `GET /api/proxy/fda/<path:fda_path>` that:
- Accepts any sub-path (e.g., `/api/proxy/fda/drug/label.json`)
- Forwards query parameters from the original request
- Injects `api_key` parameter with the `FDA_API_KEY` from environment
- Calls `https://api.fda.gov/{fda_path}?{original_params}&api_key={fda_key}`
- On success: returns the FDA response JSON
- On error: returns `{"error": "FDA data temporarily unavailable"}` with status 503
- 30-second timeout on the requests.get() call

**5. Update cloudbuild.yaml:**

Add `ALLOWED_ORIGINS=https://pharmko-app-460663669978.us-central1.run.app` to the `--set-env-vars` line in the deploy step. This ensures CORS is strict in production.

**Do NOT add any new pip dependencies** — `requests` is already available.

```python
ADMIN_EMAIL = os.environ.get("ADMIN_EMAIL", "").strip().lower()
```

**2. Create require_admin decorator (SEC-02):**

Add a `require_admin` decorator that:
- First runs the same auth check as `require_auth` (checks Bearer token, validates session)
- Then checks if `request.current_user["email"].lower() == ADMIN_EMAIL`
- If ADMIN_EMAIL is not set, returns 403 with `{"error": "Admin access not configured"}`
- If user is not authenticated, returns 401 with `{"error": "Authentication required"}`
- If user is authenticated but not admin, returns 403 with `{"error": "Admin access required"}`
- Per CONTEXT.md decision: return JSON 401/403 error, no redirect

Per Claude's Discretion in CONTEXT.md: Use 403 for "not admin" (honest — security through obscurity not needed since these are API endpoints, not pages).

**3. Protect /api/auth/toggle (SEC-02):**

Add `@require_admin` decorator to the `toggle_auth()` function.

**4. Protect /api/auth/users (SEC-02):**

Add `@require_admin` decorator to the `list_users()` function.

**5. Restrict /api/gcloud/sql/query (SEC-03):**

Add `@require_admin` decorator to the `gcloud_sql_query()` function.

Additionally, enforce a result size limit (per Claude's Discretion): add `LIMIT 1000` to the query if no LIMIT clause is present. This prevents accidental table dumps:

Keep the existing write-protection check (blocks DROP, DELETE, UPDATE, INSERT, TRUNCATE, ALTER).
