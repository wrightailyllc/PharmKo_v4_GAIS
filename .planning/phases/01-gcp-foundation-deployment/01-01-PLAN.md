Make three groups of changes to gcloud_services.py:

**Group A: ADC credential fallback**
1. In `_get_credentials()`: Replace the `raise ValueError(...)` else branch with an ADC fallback:
   ```python
   else:
       import google.auth
       _credentials, _ = google.auth.default()
       logger.info("Loaded credentials via Application Default Credentials (ADC)")
   ```
2. In `is_storage_configured()`: Remove the check for `GOOGLE_APPLICATION_CREDENTIALS_JSON` or `GOOGLE_APPLICATION_CREDENTIALS`. On Cloud Run, ADC provides credentials without these env vars. Instead, check if credentials can be obtained (try `_get_credentials()` in a try/except) AND `GCS_BUCKET_NAME` is set. Simplified logic:
   ```python
   def is_storage_configured() -> bool:
       try:
           bucket_name = os.environ.get("GCS_BUCKET_NAME")
           if not bucket_name:
               return False
           _get_credentials()  # Will succeed on Cloud Run via ADC
           return True
       except Exception:
           return False
   ```
3. In `is_sql_configured()`: Same pattern -- remove the explicit credentials file check, rely on `_get_credentials()` succeeding:
   ```python
   def is_sql_configured() -> bool:
       try:
           instance = os.environ.get("CLOUD_SQL_INSTANCE")
           user = os.environ.get("CLOUD_SQL_USER")
           password = os.environ.get("CLOUD_SQL_PASSWORD")
           database = os.environ.get("CLOUD_SQL_DATABASE")
           if not all([instance, user, password, database]):
               return False
           _get_credentials()  # Will succeed on Cloud Run via ADC
           return True
       except Exception:
           return False
   ```
4. In `get_sql_engine()`: On Cloud Run, the Connector should use ADC automatically. Update the Connector instantiation to only pass credentials when they are explicit (not ADC). Check if _credentials is from service_account module; if from google.auth.default(), pass `credentials=None` to let Connector use its own ADC detection. Simpler approach: always pass the credentials from `_get_credentials()` since `google.auth.default()` returns valid credentials the Connector can use.

**Group B: PostgreSQL DDL fixes**
5. In `initialize_cache_table()`: Replace the entire CREATE TABLE DDL with PostgreSQL-compatible syntax:
   - `id SERIAL PRIMARY KEY` (not `INT AUTO_INCREMENT PRIMARY KEY`)
   - `cached_results JSONB NOT NULL` (not `JSON NOT NULL`)
   - Remove `ON UPDATE CURRENT_TIMESTAMP` from updated_at (PostgreSQL doesn't support this -- handle in application)
   - Remove `INDEX idx_drug_name (drug_name), INDEX idx_created_at (created_at)` (PostgreSQL uses separate CREATE INDEX statements)
   - Remove `ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci`
   - Add separate CREATE INDEX statements after the CREATE TABLE

6. In `save_query_cache()`: Replace `ON DUPLICATE KEY UPDATE` with PostgreSQL's `ON CONFLICT ... DO UPDATE SET`:
   ```python
   upsert_query = """
   INSERT INTO drug_query_cache (drug_name, query_hash, cached_results, adverse_events_count, journal_articles_count)
   VALUES (:drug_name, :hash, :results, :adverse_count, :articles_count)
   ON CONFLICT (query_hash) DO UPDATE SET
       cached_results = EXCLUDED.cached_results,
       adverse_events_count = EXCLUDED.adverse_events_count,
       journal_articles_count = EXCLUDED.journal_articles_count,
       updated_at = CURRENT_TIMESTAMP
   """
   ```
   Note: Use `EXCLUDED.column_name` syntax (not `:param` references in the UPDATE clause). Remove the params dict references for the UPDATE side -- EXCLUDED references the row being inserted.

7. In `get_cached_query()`: Replace `DATEDIFF(NOW(), created_at) as days_old` with PostgreSQL equivalent:
   ```python
   EXTRACT(DAY FROM AGE(NOW(), created_at)) as days_old
   ```

8. In `get_sql_engine()` line 370: Change the default from `"mysql"` to `"postgresql"`:

```python
@app.route("/api/auth/google", methods=["GET"])
def google_auth():
    """Get Google OAuth authorization URL"""
    if not AUTH_SERVICE:
        return jsonify({"error": "Authentication service not available"}), 503

    redirect_uri = request.args.get("redirect_uri")
    if not redirect_uri:
        # Use the request's own host (works on Cloud Run, local dev, any domain)
        redirect_uri = f"{request.scheme}://{request.host}/auth/google/callback"

    auth_url = AUTH_SERVICE.get_google_auth_url(redirect_uri)
    if auth_url:
        return jsonify({"auth_url": auth_url}), 200
    return jsonify({"error": "Google OAuth not configured"}), 503
```

Key change: Remove `os.environ.get("REPLIT_DEV_DOMAIN", "")` and use `request.scheme` + `request.host` instead. This auto-adapts to whatever domain the app is running on (Cloud Run URL, localhost, future custom domain).
