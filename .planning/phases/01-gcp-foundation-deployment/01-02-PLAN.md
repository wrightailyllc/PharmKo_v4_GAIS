<task type="auto">
  <name>Task 1: Rewrite cloudbuild.yaml for Artifact Registry and secret injection</name>
  <files>cloudbuild.yaml</files>
  <action>
Completely rewrite cloudbuild.yaml. The new file should have exactly 4 steps:

**Step 1: Build container image** with Artifact Registry tags:
```yaml
- name: 'gcr.io/cloud-builders/docker'
  args:
    - 'build'
    - '-t'
    - 'us-central1-docker.pkg.dev/$PROJECT_ID/pharmko-repo/pharmko-app:$COMMIT_SHA'
    - '-t'
    - 'us-central1-docker.pkg.dev/$PROJECT_ID/pharmko-repo/pharmko-app:latest'
    - '.'
```

**Step 2: Push commit SHA tag:**
```yaml
- name: 'gcr.io/cloud-builders/docker'
  args: ['push', 'us-central1-docker.pkg.dev/$PROJECT_ID/pharmko-repo/pharmko-app:$COMMIT_SHA']
```

**Step 3: Push latest tag:**
```yaml
- name: 'gcr.io/cloud-builders/docker'
  args: ['push', 'us-central1-docker.pkg.dev/$PROJECT_ID/pharmko-repo/pharmko-app:latest']
```

**Step 4: Deploy to Cloud Run** with ALL configuration:
```yaml
- name: 'gcr.io/google.com/cloudsdktool/cloud-sdk'
  entrypoint: gcloud
  args:
    - 'run'
    - 'deploy'
    - 'pharmko-app'
    - '--image'
    - 'us-central1-docker.pkg.dev/$PROJECT_ID/pharmko-repo/pharmko-app:$COMMIT_SHA'
    - '--region'
    - 'us-central1'
    - '--platform'
    - 'managed'
    - '--allow-unauthenticated'
    - '--service-account'
    - 'pharmko-cloudrun-sa@$PROJECT_ID.iam.gserviceaccount.com'
    - '--set-secrets'
    - 'GEMINI_API_KEY=gemini-api-key:latest,FDA_API_KEY=fda-api-key:latest,GOOGLE_OAUTH_CLIENT_ID=google-oauth-client-id:latest,GOOGLE_OAUTH_CLIENT_SECRET=google-oauth-client-secret:latest,FACEBOOK_APP_ID=facebook-app-id:latest,FACEBOOK_APP_SECRET=facebook-app-secret:latest,CLOUD_SQL_PASSWORD=cloud-sql-password:latest,GCS_BUCKET_NAME=gcs-bucket-name:latest'
    - '--set-env-vars'
    - 'CLOUD_SQL_INSTANCE=$PROJECT_ID:us-central1:pharmko-db,CLOUD_SQL_USER=pharmko,CLOUD_SQL_DATABASE=pharmko,CLOUD_SQL_TYPE=postgresql,AUTH_ENABLED=true'
    - '--memory'
    - '512Mi'
    - '--timeout'
    - '300'
    - '--min-instances'
    - '0'
    - '--max-instances'
    - '10'
```

**Remove entirely:**
- The `substitutions:` block (_GEMINI_API_KEY, _FDA_API_KEY) -- secrets come from Secret Manager now
- The "create secrets" build step -- secrets already exist and should not be created during builds

**Keep:**
- `options: logging: CLOUD_LOGGING_ONLY`

**Update images section** to use Artifact Registry:
```yaml
images:
  - 'us-central1-docker.pkg.dev/$PROJECT_ID/pharmko-repo/pharmko-app:$COMMIT_SHA'
  - 'us-central1-docker.pkg.dev/$PROJECT_ID/pharmko-repo/pharmko-app:latest'
```

Add a comment at the top of the file explaining the pipeline and noting that scripts/gcp-setup.sh must be run once before the first build.

<task type="auto">
  <name>Task 2: Create GCP infrastructure setup script</name>
  <files>scripts/gcp-setup.sh</files>
  <action>
Create a new file `scripts/gcp-setup.sh` that contains all one-time GCP setup commands. This script is meant to be run manually by the user with their gcloud CLI already authenticated.

The script should:

1. **Set PROJECT_ID** at the top: `PROJECT_ID="pharmawatch-project1"` (the actual project from STATE.md)
2. **Set the project**: `gcloud config set project $PROJECT_ID`
3. **Enable required APIs** (idempotent -- safe to re-run):
   ```bash
   gcloud services enable \
     run.googleapis.com \
     cloudbuild.googleapis.com \
     artifactregistry.googleapis.com \
     secretmanager.googleapis.com \
     sqladmin.googleapis.com \
     iam.googleapis.com
   ```
4. **Create Artifact Registry repository** (check if exists first):
   ```bash
   if ! gcloud artifacts repositories describe pharmko-repo --location=us-central1 &>/dev/null; then
     gcloud artifacts repositories create pharmko-repo \
       --repository-format=docker \
       --location=us-central1 \
       --description="PharmKo container images"
   fi
   ```
5. **Create dedicated service account** (check if exists first):
   ```bash
   if ! gcloud iam service-accounts describe pharmko-cloudrun-sa@$PROJECT_ID.iam.gserviceaccount.com &>/dev/null; then
     gcloud iam service-accounts create pharmko-cloudrun-sa \
       --display-name="PharmKo Cloud Run SA"
   fi
   ```
6. **Grant IAM roles** to the service account (idempotent):
   - roles/secretmanager.secretAccessor
   - roles/cloudsql.client
   - roles/storage.objectAdmin
   - roles/logging.logWriter
7. **Grant Cloud Build permissions** to deploy to Cloud Run:
   ```bash
   PROJECT_NUMBER=$(gcloud projects describe $PROJECT_ID --format='value(projectNumber)')
   gcloud projects add-iam-policy-binding $PROJECT_ID \
     --member="serviceAccount:${PROJECT_NUMBER}@cloudbuild.gserviceaccount.com" \
     --role="roles/run.admin"
   gcloud projects add-iam-policy-binding $PROJECT_ID \
     --member="serviceAccount:${PROJECT_NUMBER}@cloudbuild.gserviceaccount.com" \
     --role="roles/iam.serviceAccountUser"
   ```
8. **Create Cloud SQL instance** (check if exists first):
   ```bash
   if ! gcloud sql instances describe pharmko-db &>/dev/null; then
     echo "Creating Cloud SQL PostgreSQL instance (this takes 5-10 minutes)..."
     gcloud sql instances create pharmko-db \
       --database-version=POSTGRES_15 \
       --tier=db-f1-micro \
       --region=us-central1 \
       --storage-size=10GB \
       --storage-type=HDD
   fi
   ```
   NOTE: Do NOT include --root-password in the script. Print a message telling the user to set the password separately: `gcloud sql users set-password postgres --instance=pharmko-db --password=YOUR_PASSWORD_HERE`
9. **Create database and user** (check if exists):
   ```bash
   gcloud sql databases create pharmko --instance=pharmko-db 2>/dev/null || echo "Database 'pharmko' already exists"
   ```
   Print a message for user to create the SQL user and store the password as a secret.
10. **List existing secrets** to help the user verify naming:
    ```bash
    echo "Existing secrets in Secret Manager:"
    gcloud secrets list --format="table(name)"
    echo ""
    echo "The app expects these secret names: gemini-api-key, fda-api-key, google-oauth-client-id, google-oauth-client-secret, facebook-app-id, facebook-app-secret, cloud-sql-password, gcs-bucket-name"
    echo "Verify the names above match or create/rename as needed."
    ```

Make the script:
- Executable (chmod instructions in a comment at the top)
- Well-commented with section headers
- Idempotent (safe to run multiple times)
- Use `set -euo pipefail` for safety
- Print status messages for each step
  </action>
