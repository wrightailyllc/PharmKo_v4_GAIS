<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Run GCP setup script and trigger first deployment</name>
  <action>
Before Claude can verify the deployment, you need to:

1. **Run the GCP setup script** (created in Plan 02):
   ```bash
   chmod +x scripts/gcp-setup.sh
   ./scripts/gcp-setup.sh
   ```
   This creates the Artifact Registry repo, service account, IAM bindings, and Cloud SQL instance. It will print status for each step and list existing secrets for you to verify naming.

2. **Verify/create secrets** in Secret Manager:
   The script will list existing secrets. Ensure these secret names exist and have correct values:
   - `gemini-api-key`
   - `fda-api-key`
   - `google-oauth-client-id`
   - `google-oauth-client-secret`
   - `facebook-app-id`
   - `facebook-app-secret`
   - `cloud-sql-password`
   - `gcs-bucket-name` (optional, only if using Cloud Storage)

   If a secret name doesn't match, create it:
   ```bash
   echo -n "YOUR_VALUE" | gcloud secrets create SECRET_NAME --data-file=-
   ```

3. **Create Cloud SQL user** (if not done by setup script):
   ```bash
   gcloud sql users create pharmko --instance=pharmko-db --password=YOUR_SECURE_PASSWORD
   ```
   Then store the password as the `cloud-sql-password` secret:
   ```bash
   echo -n "YOUR_SECURE_PASSWORD" | gcloud secrets create cloud-sql-password --data-file=-
   ```

4. **Set up Cloud Build trigger** (if not already configured):
   ```bash
   gcloud builds triggers create github \
     --repo-name=PharmKo_v4_GAIS \
     --repo-owner=YOUR_GITHUB_USERNAME \
     --branch-pattern="^main$" \
     --build-config=cloudbuild.yaml
   ```
   Or set up via Google Cloud Console > Cloud Build > Triggers.

5. **Trigger first deployment** by pushing to main or running manually:
   ```bash
   gcloud builds submit --config=cloudbuild.yaml .
   ```

6. **Get the Cloud Run URL** after deployment:
   ```bash
   gcloud run services describe pharmko-app --region=us-central1 --format='value(status.url)'
   ```

  <name>Task 2: Verify deployment health</name>
  <files></files>
  <action>
After the user provides the Cloud Run URL, verify the deployment is healthy by hitting the key endpoints:

1. **Health check:**
   ```bash
   curl -s https://CLOUD_RUN_URL/api/health | python3 -m json.tool
   ```
   Expected: `{"status": "healthy"}` with HTTP 200.

2. **Config check:**
   ```bash
   curl -s https://CLOUD_RUN_URL/api/config | python3 -m json.tool
   ```
   Expected: `ready: true`, `gcloud.sql_available: true`, `auth.database_ready: true`, `auth.google_oauth: true`, `auth.facebook_oauth: true`.

3. **SQL connection test:**
   ```bash
   curl -s https://CLOUD_RUN_URL/api/gcloud/sql/test | python3 -m json.tool
   ```
   Expected: `connected: true`.

4. **Frontend loads:**
   ```bash
   curl -s -o /dev/null -w "%{http_code}" https://CLOUD_RUN_URL/
   ```
   Expected: HTTP 200.

If any check fails, examine Cloud Run logs:
```bash
gcloud run services logs read pharmko-app --region=us-central1 --limit=50
```

Report results and any issues to the user.

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 3: Update OAuth provider consoles with Cloud Run URL</name>
  <action>
The Cloud Run URL is now known. You need to register it with Google and Facebook OAuth:

**Google OAuth Console:**
1. Go to: Google Cloud Console > APIs & Services > Credentials
2. Click on the OAuth 2.0 Client ID used by PharmKo
3. Under "Authorized redirect URIs", add:
   `https://CLOUD_RUN_URL/auth/google/callback`
4. Under "Authorized JavaScript origins", add:
   `https://CLOUD_RUN_URL`
5. Save

**Facebook OAuth Console:**
1. Go to: Facebook Developer Console > Your App > Facebook Login > Settings
2. Under "Valid OAuth Redirect URIs", add:
   `https://CLOUD_RUN_URL/auth/facebook/callback`
3. Save

After saving both, verify by testing Google OAuth login on the deployed app.
  </action>
  <resume-signal>Type "done" after updating both OAuth consoles, or describe any issues.</resume-signal>
</task>
